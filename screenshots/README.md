# Screenshots Policy

This folder is reserved for curated, sanitized screenshots of ABYSS.

## Objective

Show visual depth without exposing:

- personal data
- student records
- billing identifiers
- support content
- internal audit traces
- sensitive operational details

## Recommended screenshot set

1. `01-login-shell.png`
2. `02-internal-dashboard-sanitized.png`
3. `03-courses-or-students-sanitized.png`
4. `04-student-portal-preview.png`
5. `05-student-documents-preview.png`
6. `06-sgc-dashboard-sanitized.png`

## Rules

- Never capture live personal data.
- Prefer preview or demo flows where available.
- If an internal module must be shown, scrub names, emails, phone numbers, IDs, invoice numbers, and operational comments.
- Keep captures focused on layout, navigation, components, and workflow depth.
- Avoid screenshots that reveal infrastructure details, browser secrets, or internal URLs beyond the public product domain.
