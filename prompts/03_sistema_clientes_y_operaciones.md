# Prompt — Sistema de Clientes, Leads, Alumnos y Operación

```md
Actúa como product architect de ABYSS.

Quiero fortalecer y documentar la parte de gestión de clientes dentro de ABYSS, entendiendo "cliente" en sentido amplio:
- lead
- potencial alumno
- alumno
- antiguo alumno
- contacto de soporte
- comprador / facturación

No lo enfoques como CRM genérico. Enfócalo como sistema operativo comercial-académico dentro de un SaaS multirol.

Necesito que entregues:

1. Modelo de ciclo de vida:
- lead
- lead cualificado
- convertido
- alumno activo
- alumno recurrente
- alumno egresado
- alumno con soporte / deuda / documentación pendiente

2. Diseño de dominio:
- entidades clave
- relaciones
- campos críticos
- ownership entre ventas, coordinación, instructor y alumno

3. Flujos críticos:
- captación
- seguimiento
- conversión
- inscripción
- onboarding documental
- soporte
- reactivación

4. Integraciones:
- WhatsApp
- pagos
- facturación
- portal alumno
- documentación SGC

5. Documentación prioritaria:
- mapa lifecycle
- modelo de datos
- mapa de ownership por rol
- flujos end-to-end
- pantallas clave
- evidencias para repositorio público

6. Qué genera más impacto:
- para CTO
- para hiring manager
- para evaluación de producto
- para due diligence

Formato:
- Resumen
- Modelo lifecycle
- Arquitectura funcional
- Roles y ownership
- Integraciones
- Riesgos
- Recomendación de documentación priorizada
```
